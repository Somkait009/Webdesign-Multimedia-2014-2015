CSS

Entwicklung

CSS: "Cascading Style Sheet" ist eine Beschreibungssprache für Stylesheets. Sie dient als "Formatvorlage" für HTML-Seiten und wurde im Dezember 1996 vom W3C herausgegeben. Der Vorgänger von CSS war "Cascading HTML Style Sheet" (CHSS), welcher im Oktober 1994 von Håkon Wium Lie vorgeschlagen wurde.

Syntax

Stylesheet

Als Stylesheet wird eine Datei oder ein Abschnitt innerhalb eines Dokuments bezeichnet, der aus CSS-Regelsätzen besteht.

Regelsätze

In CSS sind Eigenschaften innerhalb von Regelsätzen organisiert. Ein Regelsatz besteht aus:

  • Einem Selektor oder einer Gruppe von Selektoren, gefolgt von
  • einem durch geschweifte Klammern begrenzten Bereich, in dem Eigenschaften ein Wert zugewiesen wird. 
Selektor {
    Eigenschaft1: Wert;
    Eigenschaft2: Wert;
}

Selektor Eins, Selektor Zwei {
    Eigenschaft: Wert;
}

Hier zwei Beispiele für einen Regelsatz. Der erste Regelsatz besteht aus einem Selektor, dem zwei Eigenschaften zugewiesen werden. Der zweite Regelsatz besteht aus zwei voneinander unabhängigen Selektoren, denen dieselbe Eigenschaft zugewiesen wird. Ein Format, das mehrere Elemente eines Dokuments anspricht, muss daher nur einmal definiert werden.

Ein Stylesheet kann schon durch das Vorhandensein einer einzigen Regel entstehen.

Kommentare

Um die Übersicht zu behalten oder bestimmte Regelsätze zu erläutern ist es möglich, innerhalb von Stylesheets Kommentare zu verfassen. Ein Kommentar beginnt mit den Zeichen „/*“ und endet mit dem ersten Auftreten der Zeichen „*/“. Das bedeutet, dass Kommentare nicht ineinander verschachtelt werden können.

/* Dies ist ein Kommentar. Er besitzt keine Auswirkungen. */
/* Verschachtelte /* Kommentare sind */ nicht möglich */

Kommentare dürfen sich über mehrere Zeilen erstrecken. Jedoch ist es selbst durch das mehrfache Auftreten der Zeichen „/*“ nicht möglich, Kommentare zu verschachteln.
Beachten Sie: Die aus vielen Programmiersprachen bekannten einzeiligen Kommentare gibt es in CSS nicht.

Eigenschaften und Werte (Deklaration)

Bereits im Abschnitt „Regelsätze“ haben Sie das allgemeine Aussehen einer Deklaration (also einer Eigenschaft-Wert-Zuordnung) gesehen: Dem Namen der Eigenschaft folgt ein Doppelpunkt, dem wiederum der Wert folgt, der der Eigenschaft zugewiesen wird. Es gibt Eigenschaften, denen kann nicht nur ein Wert, sondern mehrere Werte zugewiesen werden. In diesen Fällen werden die einzelnen Werte durch ein Leerzeichen voneinander getrennt.

Selektor {
    Eigenschaft: Wert 1.0;
}

Dieses Beispiel zeigt eine Eigenschaft, der zwei Werte zugewiesen werden. Einmal ein Schlüsselwort („Wert“) und eine Zahl („1.0“).

„shorthand“-Eigenschaften

Werden einer Eigenschaft mehrere Werte zugewiesen, handelt es sich üblicherweise um eine so genannte „shorthand“-Eigenschaft. Dabei handelt es sich um Eigenschaften, die mehrere andere Eigenschaften zusammenfassen.

Überschrift {
    Schrift-Größe: 200%;
    Schrift-Stil: kursiv;
}

Selektor {
    Schrift: 200%
    kursiv;
}

Bei der Eigenschaft „Schrift“ handelt es sich um eine „shorthand“-Eigenschaft, die die detailierteren Eigenschaften „Schrift-Größe“ und „Schrift-Stil“ kombiniert.

Weitere Informationen zur Syntax von CSS

Beispiel

Dies ist ein Beispiel einer Klasse in einer css-Datei

example {
    color: ;
    background-color: ;
    font-size: ;
    font-style: ;
    font-weight: ;
    line-height: ;
    display: ;
}

Für die einzelnen Eigenschaften können verschiedene Werte eingetragen werden. Farben kann man entweder direkt beschreiben (z.B. red, blue, yellow) oder als hexadezimale Zahlenwerte (z.B.#ffffff, #147adf, #666). Die Schriftgröße (font-size) kann man sowohl in Pixeln angeben (z.B. 25px, 50px), als auch Prozentwerte oder direkte Bezeichner (z.B. large, small)
