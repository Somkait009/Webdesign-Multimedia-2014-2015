Hypertext Markup Language

Entwicklung

HTML: "Hypertext Markup Language" ist eine Formatierungssprache wie z.B. LaTeX, welche am 13. März 1989 von Tim Bernes-Lee festgelegt wurde. Die erste Version wurde am 3. November 1992 veröffentlicht. Diese Version orientierte sich lediglich am Text und hatte noch keine Versionsnummer. Am 30. April 1993 erhielt HTML die Fähigkeit Bilder einzubinden und Textattribute wie fett, kursiv, usw. zu verwenden. Seit dem 24. Dezember 1999 wurde HTML in der Version 4.01 weiterentwickelt und erhielt viele neue Möglichkeiten, wie Skripte, Stylesheets, Formulare, usw.

Ab 2014 steht der neue Standard HTML 5, welcher neue Funktionen unterstüzt wie Video, Audio, lokalen Speicher und dynamische 2D- und 3D-Grafiken, die von HTML4 nicht direkt unterstützt wurden und sich ohne HTML5 nur mit zusätzlichen Plugins (z. B. Adobe Flash) umsetzen ließen.

Syntax

"Befehle" werden als sogenannte Tags dargestellt, z.B. für hervorgehobenen Text:
<strong>Fett!</strong>

Kommentare werden in HTML wie folgt dargestellt:
<!-- Kommentar -->

Ein Problem von HTML ist die Darstellung von für HTML reservierten Zeichen wie z.B. <, >, oder &. Diese Sonderzeichen heißen "HTML entities".

HTML Darstellung
&lt; <
&gt; >
&amp; &

Strukturierung

            <!DOCTYPE html>
<html>
<head>
    <!-- Hier kommen "Kopfinformationen" rein. -->
    <meta ... />
    <title>Seitentitel</title>
    <!-- ... -->
</head>
<body>
    <!-- Hier steht der eigentliche Inhalt. -->
</body>
</html>

<head>...</head>
In den Kopf des Dokuments ("head") kommt alles rein, was nicht zum Inhalt der Seite gehört, wie z.B. Seitentitel, Meta-Informationen, Skripte, Stylesheets, usw.

<body>...</body>
In den Körper des Dokuments ("body") kommt der eigentliche Inhalt der Seite, wie z.B. Text, Bilder, Videos, Sound, Formulare, usw.

Textformatierung

Überschriften werden durch folgende Tags repräsentiert. “1” ist die gröbste Unterteilung, “2” sind Unter-Sektionen, “3” Unter-Unter-Sektionen, usw.
<h1>...</h1> bis <h6>...</h6>

HTML bietet verschiedene Tags, die Text verschieden darstellen.

HTML Darstellung
<b>...</b> fett
<i>...</i> kursiv
<u>...</u> unterstrichen

Intern darf der Text beliebig formatiert werden. Der Browser wird immer einzelne Leerzeichen statt mehreren oder Zeilenumbrüchen anzeigen. Zeilenumbrüche und Absätze werden durch "HTML entities" oder Tags dargestellt.

HTML Textformatierung
&nbsp; Zeilenumbruch verhindern
<br /> Zeilenumbruch erzwingen
<p>...</p> Absätze

Strukturierung

Strukturierung erfolgt mittels sogenannter Bereiche.

HTML Strukturierung
<div> Bereiche
<aside> Nebeninformationen
<header> Seitenkopf
<footer> Fußzeile
<nav> Navigation
<article> Abgeschlossener Inhalt
<section> Abschnitt

Tabellen sind für tabellarische Inhalte (z.B. wissenschaftliche Ergebnisse) und nicht etwa für die Webseitenstrukturierung gedacht.

Cell 1 Cell 2 Cell 3
Cell 4

Diese Tabelle werden durch den folgenden Code repräsentiert:

<table>
    <tr>
        <td rowspan="2">Cell 1</td>
        <td>Cell 2</td>
        <td>Cell 3</td>
    </tr>
    <tr>
        <td colspan="2">Cell 4</td>
    </tr>
</table>

Medien

Externe Dateien, z.B. Bilder, sollten möglichst relativ referenziert werden. Relative Pfade suchen in dem Ordner, in dem die Quelldatei liegt und orientieren sich an dessen Struktur. Anstatt eines absoluten Pfades, wie z.B. "C:\Users\...\images\bild.jpg" sollte ein relativer Pfad, wie "images/bild.jpg" verwendet werden. Dieser findet das Bild unabhängig vom Pfad des Ordners.

Bilder werden mit dem HTML-Tag <img> eingebunden. Das Bild "logo.png" wird im Verzeichnis "images" eingebunden. Das Attribut "alt" wird benötigt, falls das Bild nicht vorhanden ist. In diesem Fall kann man ein Fehlerbild oder einen Fehlertext darstellen.
<img src="images/logo.png" alt="Logo">
Weitere Informationen zu Attributen von Bildern

Links können unter HTML wie folgt verwendet werden:
<a href="https://www.google.de/">Google</a>

Bei einzelnen Punkten innerhalb eines Dokuments spricht man von "Ankern":
<a id="Anker"></a>
Hiermit definiert man einen Anker mit der ID "Anker". Dieser kann wie folgt referenziert werden:
<a href="#Anker">Klick mich!</a>
